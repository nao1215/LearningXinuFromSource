31 ## cpsr（カレントプログラムステータスレジスタ）

40 | 28 | V | 演算結果がオーバフローの場合に設定される |

57 ## コプロセッサ
58 
59 プロセッサキャッシュや MMU の操作などでは、コプロセッサとコプロセッサ用レジスタを使用する。<br>
60 コプロセッサは 16 個存在する。CP0〜9、CP12、CP13 はシステム独自の外部コプロセッサとして使用できる。<br>
61 その他のコプロセッサは、下表に示す通り、予め役割を定められている。

67 | CP14 | デバッグおよび ETM（Embedded Trace Macrocell、実行命令履歴）制御 |
68 | CP15 | システム制御 |

94 ## IRQ／FIQ 割り込み
95 
96 FIQは高い優先度の高速割り込みであり、IRQ は通常優先度の割り込みである。<br>
97 FIQは、プロセッサが状態を保存しなくて良いハードウェアのために存在し、
98 割り込み要求の処理中に「IRQ」および「その他の FIQ ハンドラ」を無効にする事によって<br>
99 優先順位を付けている。FIQ ハンドラ処理中に他の割り込みは発生しない。<br>
100 <br>
101 cpsr の I ビットが 0 および IRQ 端子が有効になった場合、IRQ 割り込み例外が発生する。<br>
102 cpsr の F ビットが 0 および FIQ 端子が有効になった場合、FIQ 割り込み例外が発生する。<br>

104 ## インラインアセンブラ
105 
106 C 言語コード中でアセンブラを使用でき、インラインアセンブラ使用時に用いる命令は下表の通りである。<br>
107 
108 | 命令 | 役割 |
109 | -------- | ------------------------------------------ |
110 | asm | インラインアセンブラの仕様を意味する（※3） |
111 | volatile | コードの最適化を抑制する。 |
112 
113 構文は、asm(アセンブラコード :出力オペランド :入力オペランド :上書きされるレジスタリスト)という形式で、<br>
114 ":"以降の指定は全てオプションである。記載例は以下の通りである。
115 
116 ```
117 int func(int *a)
118 {
119  int result = 0;
120 
121  asm volatile (
122  "mov r1, #99;"
123  "str r1, [%1];"
124  "mov %0, #88;"
125  : "=r"(result) /* 出力オペランド。"=r"の後に(C言語の変数名)を記載する */
126  : "r"(a) /* 入力オペランド・"=r"の後に(C言語の変数名)を記載する */
127  : /* オプションなので無記載でも良い。記載する場合は"r2"、"r3"...と記載する。*/
128  ); /* 補足：%n部分（nは正の数値）が入出力オペランド。数値は定義順で振られる。 */
129  return ret;
130 }
131 ```
